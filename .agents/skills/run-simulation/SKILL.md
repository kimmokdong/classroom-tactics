# Run Simulation Skill

## Trigger
유저가 "시뮬 돌려", "시뮬레이션 실행해 줘", "시뮬레이터 돌려" 등의 명령을 내릴 때 이 스킬을 실행하세요.

## Instructions
이 스킬이 발동되면 다음 단계를 **반드시 순서대로** 수행하세요.

1. **시뮬레이션 실행**
   - `run_command` 툴을 사용하여 `c:\Users\hyunseung\Desktop\classroom-tactics` 경로에서 `node run_simulation.js` 명령어를 실행합니다.
   - 툴 호출 시 백그라운드 태스크(Task ID)가 완료될 때까지 기다리세요. (manage_task 활용)

2. **보고서 양식 기반 마크다운 텍스트 생성 (핵심 규칙)**
   - 시뮬레이션 결과를 읽고 분석할 때, **반드시 아래의 목차와 템플릿 구조를 100% 동일하게 지켜서** 전체 마크다운 텍스트를 작성해야 합니다. 목차 하나라도 누락해서는 안 됩니다.

3. **아티팩트 업데이트 (UI 연동)**
   - 생성된 마크다운 텍스트를 반드시 **`write_to_file` 또는 `multi_replace_file_content` 툴**을 사용하여 아티팩트에 직접 덮어씌워야 합니다.
   - `TargetFile`: `C:\Users\hyunseung\.gemini\antigravity-ide\brain\<현재 대화 ID>\latest_simulation_report.md`
   - `IsArtifact`: `true`
   - `Overwrite`: `true`
   - `ArtifactMetadata`: `{"ArtifactType": "other", "RequestFeedback": false, "Summary": "최신 시뮬레이션 분석 리포트"}`
   - **절대로** bash 명령어(`fs.writeFileSync` 등)로 아티팩트 파일을 덮어쓰지 마세요. 

4. **완료 보고**
